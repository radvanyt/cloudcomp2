REST API resources:

GET USERS/{ID}                || returns user ID and username                    || SELECT in USER table
GET USERS/{USERNAME}            

POST USERS/{ID}               || sends a message to a specific user              ||INSERT into MESSAGES table, and into REIEVED table
POST USERS/ALL                || broadcasts a message to all users               || --,-- multiple inserts to REC. table

GET USERS/{ID}/RECIEVED       || returns all messages sent to the user with {ID} || select on REC. table, returning IDs (possible join for timestamp, sender), update boolean
GET USERS/{ID}/SENT           || returns the message IDs sent by the user        || select on MESSAGES
GET MESSAGES/{ID}             || returns a specific message                      || SELECT and JOIN on REC, and MESSAGES (read info, text, tmstmp, sender)
DELETE MESSAGES/{ID}          || delete a specific message (if noone have read it)| check if boolean is true, if false 4 everybody, delete MESSAGES,  and REC

PUT USERS/{ID}                || to change username info                         || update in USERS
POST USERS/                   || create user                                     || INSERT in USERS
